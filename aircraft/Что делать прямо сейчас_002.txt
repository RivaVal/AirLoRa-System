## 📡 3.2 & 3.3. Как реализовать пайплайн (стрим + архив + оверлей + ESP32)

На Pi Zero 2W **нельзя одновременно** выдавать `4K@30fps` и делать `12МП` снимки без просадок. Но `picamera2` решает это через **мульти-поточную конфигурацию**:

```python
# camera_pipeline.py
import cv2
import time
import socket
import numpy as np
from picamera2 import Picamera2
from libcamera import controls
import os

class CameraPipeline:
    def __init__(self, esp_ip="192.168.4.2", esp_port=5000):
        self.picam2 = Picamera2()
        self.esp_ip = esp_ip
        self.esp_port = esp_port
        self.sock = socket.socket(socket.AF_INET, socket.SOCK_STREAM)
        
        # Мульти-поток: lores (для стрима), main (макс. качество для снимков)
        self.config = self.picam2.create_video_configuration(
            main={"size": (2592, 1944)},  # Макс. разрешение матрицы
            lores={"size": (640, 480)},   # Для стрима/анализа
            buffer_count=3
        )
        self.picam2.configure(self.config)
        self.picam2.start()
        
        # Подключение к ESP32-S3
        try:
            self.sock.connect((self.esp_ip, self.esp_port))
            print(f"✅ Подключено к ESP32-S3: {self.esp_ip}:{self.esp_port}")
        except Exception as e:
            print(f"⚠️ ESP32 недоступен: {e}. Работаем в локальном режиме.")

    def capture_max_quality(self, filename="archive/frame.jpg"):
        os.makedirs("archive", exist_ok=True)
        self.picam2.capture_file(filename)
        print(f"📸 Сохранён кадр макс. качества: {filename}")

    def send_stream_frame(self, telemetry=None):
        # Получаем кадр низкого разрешения для стрима
        frame = self.picam2.capture_array("lores")
        
        # Накладываем телеметрию по запросу
        if telemetry:
            cv2.putText(frame, f"TELEM: {telemetry}", (10, 30),
                        cv2.FONT_HERSHEY_SIMPLEX, 0.6, (0, 255, 0), 2)
            cv2.putText(frame, time.strftime("%H:%M:%S"), (10, 60),
                        cv2.FONT_HERSHEY_SIMPLEX, 0.5, (0, 255, 255), 2)
        
        # Кодируем в JPEG и отправляем на ESP32
        _, jpeg = cv2.imencode(".jpg", frame, [cv2.IMWRITE_JPEG_QUALITY, 85])
        try:
            self.sock.sendall(b"FRAME" + len(jpeg).to_bytes(4, "big") + jpeg.tobytes())
        except Exception:
            pass  # ESP32 не в сети, не падаем

        # Сохраняем в архив (опционально)
        with open("archive/stream_latest.jpg", "wb") as f:
            f.write(jpeg.tobytes())

    def run(self):
        print("🚀 Запуск пайплина. Ctrl+C для остановки.")
        try:
            while True:
                self.send_stream_frame(telemetry="T:24C V:3.7V")
                # Периодический снимок макс. качества (каждые 15 сек)
                if int(time.time()) % 15 == 0:
                    self.capture_max_quality()
                time.sleep(1/15)  # ~15 FPS стрим
        except KeyboardInterrupt:
            print("\n🛑 Остановка...")
        finally:
            self.sock.close()
            self.picam2.stop()

if __name__ == "__main__":
    cam = CameraPipeline(esp_ip=os.getenv("ESP32_IP", "192.168.4.2"))
    cam.run()
```

## 🧭 4.2. Архитектура: `ESP32-S3` + `Pi Zero 2W` + `G4/5G`
Твоя схема **логична и соответствует индустриальным стандартам** для автономных дронов. Разделение ответственности правильное:

| Компонент | Роль | Почему это хорошо |
|-----------|------|-------------------|
| `ESP32-S3` | Полётный контроллер, сбор телеметрии, управление периферией | Детерминированный RTOS, низкая задержка прерываний, низкое энергопотребление |
| `Pi Zero 2W` | Захват CSI, обработка видео, оверлей телеметрии, кодирование, стриминг | 4× Cortex-A53, аппаратный ISP, поддержка `libcamera`, Linux-стек |
| `G4/5G модем` | Дальнобойный канал передачи архива/телеметрии/бэкап-потока | Глобальное покрытие, высокая пропускная способность |

### ⚠️ Где возникают сомнения и как их закрыть:
1. **ESP32-S3 и "обработка видео"**  
   У ESP32-S3 максимум 8 МБ PSRAM и нет аппаратного видео-энкодера. Кодировать `1080p@30fps` или накладывать графику он **не потянет** без сильных просадок и зависаний полётного цикла.  
   ✅ **Решение:** Оставь всю видеологик на Pi. ESP32 отправляет телеметрию по `UART` (1.5–3 Мбит/с), Pi принимает, рисует поверх кадра и отдаёт в стрим/архив.

2. **Связь ESP32 ↔ Pi**  
   `I2C` слишком медленный для потока телеметрии. `SPI` быстр, но требует синхронизации по тактам.  
   ✅ **Решение:** `UART` на `921600` или `1.5 Мбит/с` с бинарным протоколом (например, `MAVLink` или простой `TLV`). Этого достаточно для 50-100 Гц телеметрии.

3. **G4/5G для непрерывной трансляции**  
   Сотовые сети дают `jitter` 50-300 мс и задержку 100-500 мс. Для пилота в режиме реального времени это **неприемлемо** (нужно <50 мс).  
   ✅ **Решение:**  
   - `5.8/5 GHz WiFi` (прямое соединение) → низкая задержка для FPV-пилотажа  
   - `G4/5G` → архивация, телеметрия, бэкап-поток, передача на сервер после посадки

4. **Энергопотребление и вес**  
   Pi Zero 2W (~200 мА) + камера (~250 мА) + G4 модем (до 800 мА в передаче) + АКБ.  
   ✅ **Решение:** Используй импульсные стабилизаторы (DC-DC), отключай модем на взлёте/посадке, ставь `ondemand` governor на Pi. Вес всей видеосвязки ~45-60 г (без АКБ).

👉 **Итог по архитектуре:** Меняй только распределение задач. `ESP32-S3` → полёт + датчики + отправка телеметрии по UART. `Pi Zero 2W` → камера → overlay → кодирование → `5GHz WiFi` (live) + `G4/5G` (archive). Архитектура рабочая.

## 📹 3.2 & 3.3. Реализация: стрим + макс. качество + оверлей телеметрии

`picamera2` нативно поддерживает **два независимых выхода** на одном сенсоре:
- `main` → полное разрешение (для снимков по запросу)
- `lores` → уменьшенный поток (для кодирования/стрима)

### Минимальный рабочий пайплайн (`camera_pipeline.py`)
```python
import cv2
import time
import serial
import os
from picamera2 import Picamera2
from libcamera import controls

class DroneCamera:
    def __init__(self, uart_port="/dev/ttyAMA0", baud=921600):
        self.picam2 = Picamera2()
        # Двухпоточная конфигурация
        self.config = self.picam2.create_video_configuration(
            main={"size": (2592, 1944)},   # Макс. разрешение (для снимков)
            lores={"size": (1280, 720)},   # Для стрима/оверлея
            buffer_count=3
        )
        self.picam2.configure(self.config)
        self.picam2.start()
        
        # UART к ESP32
        self.uart = serial.Serial(uart_port, baud, timeout=0.1)
        self.telemetry = {}
        
    def read_telemetry(self):
        """Читает строку вида 'TEMP=24.5;BAT=3.7;ALT=12.3' от ESP32"""
        try:
            data = self.uart.readline().decode("utf-8").strip()
            if 
                self.telemetry = dict(kv.split("=") for kv in data.split(";") if "=" in kv)
        except Exception:
            pass

    def capture_max_quality(self, folder="archive"):
        os.makedirs(folder, exist_ok=True)
        fn = f"{folder}/HQ_{int(time.time())}.jpg"
        self.picam2.capture_file(fn)
        return fn

    def stream_frame_with_overlay(self, show_telem=False):
        frame = self.picam2.capture_array("lores")
        if show_telem and self.telemetry:
            text = " | ".join(f"{k}:{v}" for k, v in self.telemetry.items())
            cv2.putText(frame, text, (10, 30), cv2.FONT_HERSHEY_SIMPLEX, 0.6, (0,255,0), 2)
            cv2.putText(frame, time.strftime("%H:%M:%S"), (10, 60), cv2.FONT_HERSHEY_SIMPLEX, 0.5, (0,255,255), 2)
        return frame

    def run(self):
        print("🚀 Камера запущена. Ctrl+C для остановки.")
        try:
            while True:
                self.read_telemetry()
                frame = self.stream_frame_with_overlay(show_telem=True)
                # Здесь отправляй `frame` в ffmpeg/gstreamer для стрима или записи
                time.sleep(1/30)  # ~30 FPS стрим
        except KeyboardInterrupt:
            pass
        finally:
            self.picam2.stop()
            self.uart.close()

if __name__ == "__main__":
    DroneCamera().run()
```

🔹 **Как отдавать в стрим/архив:**  
Передавай `frame` в `ffmpeg` через `pipe:` или используй `picamera2` напрямую с `output="file.mp4"` + `output="udp://IP:PORT"`. Архив пишется в примонтированную папку `/workspace/archive`.

## 📡 2. Бинарный протокол телеметрии ESP32 → Pi (UART)

Простой, помехоустойчивый, с проверкой целостности. Использует `struct` + синхробайты + CRC16.

### 📘 `telemetry_protocol.h` (общий для ESP32 и Pi)
```cpp
#pragma once
#include <cstdint>

#define TELEMETRY_SYNC1 0xAA
#define TELEMETRY_SYNC2 0x55

#pragma pack(push, 1)
struct TelemetryFrame {
    uint8_t sync1 = TELEMETRY_SYNC1;
    uint8_t sync2 = TELEMETRY_SYNC2;
    uint16_t seq;
    uint32_t timestamp_ms;
    float altitude_m;
    float battery_v;
    float cpu_temp_c;
    uint8_t gps_fix;
    uint8_t flags; // биты: 0=armed, 1=streaming, 2=low_bat
    uint16_t crc16;
};
#pragma pack(pop)

// Простая CRC16-CCITT (табличная или полиномиальная)
inline uint16_t calc_crc16(const uint8_t* data, size_t len) {
    uint16_t crc = 0xFFFF;
    for (size_t i = 0; i < len; i++) {
        crc ^= (uint16_t)data[i] << 8;
        for (int j = 0; j < 8; j++) {
            if (crc & 0x8000) crc = (crc << 1) ^ 0x1021;
            else crc <<= 1;
        }
    }
    return crc;
}
```

### 📤 Отправитель (ESP32, ESP-IDF / Arduino)
```cpp
#include "telemetry_protocol.h"
#include <HardwareSerial>

HardwareSerial TelemetrySerial(2); // UART2: TX=17, RX=16
uint16_t seq = 0;

void sendTelemetry(float alt, float bat, float temp, bool armed, bool gps_ok) {
    TelemetryFrame frame;
    frame.seq = seq++;
    frame.timestamp_ms = millis();
    frame.altitude_m = alt;
    frame.battery_v = bat;
    frame.cpu_temp_c = temp;
    frame.gps_fix = gps_ok ? 1 : 0;
    frame.flags = (armed ? 1 : 0) | (1 << 1); // streaming flag
    
    // CRC считается по всему пакету БЕЗ поля crc16
    frame.crc16 = calc_crc16((uint8_t*)&frame, sizeof(TelemetryFrame) - sizeof(frame.crc16));
    
    TelemetrySerial.write((uint8_t*)&frame, sizeof(frame));
}
// Вызывай 10-50 Гц в задаче или таймере
```

### 📥 Приёмник (Pi Zero 2W, Python)
```python
import serial
import struct
import time
from telemetry_protocol_h import TelemetryFrame, calc_crc16  # импортируй логику или продублируй

ser = serial.Serial('/dev/ttyAMA0', 921600, timeout=0.05)
buffer = b""

def parse_frame(data):
    frame = TelemetryFrame.from_buffer_copy(data)
    expected = calc_crc16(data[:-2])
    if frame.crc16 != expected:
        return None
    return frame

while True:
    buffer += ser.read(ser.in_waiting)
    while TELEMETRY_SYNC1.to_bytes(1) + TELEMETRY_SYNC2.to_bytes(1) in buffer:
        idx = buffer.find(TELEMETRY_SYNC1.to_bytes(1) + TELEMETRY_SYNC2.to_bytes(1))
        buffer = buffer[idx:]
        if len(buffer) < struct.calcsize('=BBHIfffBBH'):
            break
        raw = buffer[:struct.calcsize('=BBHIfffBBH')]
        frame = parse_frame(raw)
        if frame:
            print(f"📊 Alt:{frame.altitude_m:.1f}m Bat:{frame.battery_v:.2f}V Temp:{frame.cpu_temp_c:.1f}°C Fix:{frame.gps_fix}")
            buffer = buffer[struct.calcsize('=BBHIfffBBH'):]
        else:
            buffer = buffer[2:]  # сдвиг, если мусор
```
✅ Протокол устойчив к обрывам, битам и перезапускам. Работает на `1.5 Мбит/с` без потерь.

## 📺 3. Приём и декодирование потока на стороне оператора

Pi Zero 2W шлёт сырой `H.264` по UDP. На земле принимаешь и декодируешь.

### 🎯 Вариант 1: `ffplay` (максимально просто, задержка ~150-300 мс)
```bash
ffplay -fflags nobuffer -flags low_delay -framedrop \
       -i udp://@0.0.0.0:5000?fifo_size=1000000&overrun_nonfatal=1
```
- `nobuffer + low_delay` → минимизирует буферизацию
- `framedrop` → пропускает кадры при лагах сети, не тормозит картинку

### 🎯 Вариант 2: `gstreamer` (надежнее при джиттере, задержка ~100-200 мс)
```bash
gst-launch-1.0 -v \
  udpsrc port=5000 caps="application/x-rtp, media=video, clock-rate=90000, encoding-name=H264" ! \
  rtph264depay ! \
  h264parse ! \
  avdec_h264 ! \
  videoconvert ! \
  autovideosink sync=false
```
- `sync=false` → отключает привязку к системным часам (критично для низкой задержки)
- Если шлёшь `raw H.264` без RTP-заголовков, замени первую строку на:
  `udpsrc port=5000 ! h264parse ! avdec_h264 ! videoconvert ! autovideosink sync=false`

### 💡 На стороне Pi (отправка в поток)
В `ffmpeg` или `gstreamer` на Pi добавь:
```bash
# ffmpeg пример:
ffmpeg -f v4l2 -i /dev/video0 -c:v libx264 -preset ultrafast -tune zerolatency -b:v 2M \
       -f h264 udp://<GROUND_IP>:5000
```
- `preset ultrafast + tune zerolatency` → обязательно для FPV-задержки
- `b:v 2M` → баланс качество/сеть (для WiFi 5GHz можно 4-6М, для G4/5G → 1.5-2М)

```cpp
};
#pragma pack(pop)

inline uint16_t calc_crc16(const uint8_t* data, size_t len) {
    uint16_t crc = 0xFFFF;
    for (size_t i = 0; i < len; i++) {
        crc ^= (uint16_t)data[i] << 8;
        for (int j = 0; j < 8; j++) crc = (crc & 0x8000) ? ((crc << 1) ^ 0x1021) : (crc << 1);
    }
    return crc;
}
```

### 📜 `src/uart_telemetry.cpp`
```cpp
#include "uart_telemetry.h"
#include <fcntl.h>
#include <termios.h>
#include <unistd.h>
#include <cstring>

bool UartTelemetry::open(const std::string& port, int baudrate) {
    fd_ = ::open(port.c_str(), O_RDWR | O_NOCTTY | O_NONBLOCK);
    if (fd_ < 0) { perror("UART open"); return false; }

    termios tty{};
    tcgetattr(fd_, &tty);
    cfsetispeed(&tty, B921600);
    cfsetospeed(&tty, B921600);
    tty.c_cflag = (tty.c_cflag & ~CSIZE) | CS8;
    tty.c_cflag |= (CLOCAL | CREAD);
    tty.c_cflag &= ~(PARENB | CSTOPB | CRTSCTS);
    tty.c_lflag = 0; tty.c_oflag = 0;
    tty.c_iflag &= ~(IXON | IXOFF | IXANY | INLCR | ICRNL | IGNCR);
    tty.c_cc[VMIN] = 0; tty.c_cc[VTIME] = 1;
    tcsetattr(fd_, TCSANOW, &tty);
    tcflush(fd_, TCIOFLUSH);
    return true;
}

void UartTelemetry::close() { if (fd_ >= 0) ::close(fd_); }

bool UartTelemetry::poll(std::function<void(const TelemetryFrame&)> callback) {
    uint8_t buf[64];
    ssize_t n = read(fd_, buf, sizeof(buf));
    if (n <= 0) return false;

    for (int i = 0; i <= n - (int)sizeof(TelemetryFrame); i++) {
        if (buf[i] == TELEMETRY_SYNC1 && buf[i+1] == TELEMETRY_SYNC2) {
            TelemetryFrame frame;
            std::memcpy(&frame, &buf[i], sizeof(frame));
            if (calc_crc16((uint8_t*)&frame, sizeof(frame) - 2) == frame.crc16) {
                callback(frame);
                return true;
            }
        }
    }
    return false;
}
```

## 📦 Технологический стек для Pi Zero 2W

### Видео-пайплайн
| Компонент | Пакет/Библиотека | Зачем |
|-----------|-----------------|-------|
| Захват CSI | `libcamera` + `python3-picamera2` | Современный стек для камер RPi (замена legacy `raspistill`) |
| Обработка кадров | `python3-opencv` + `numpy` | Наложение телеметрии, детекция, сжатие |
| Кодирование | `ffmpeg` (libx264) или `gstreamer1.0` | H.264/MJPEG для стрима и архива |
| Стриминг | `ffmpeg -f h264 udp://IP:PORT` или `gstreamer rtph264pay` | Передача видео по сети с низкой задержкой |
| Архивация | Прямая запись в файл + ротация по времени/размеру | Сохранение полётных данных |

### Связь с ESP32-S3
| Компонент 	   | Реализация | Примечание |
|------------------|------------|------------|
| Аппаратный UART  | `/dev/ttyAMA0` (Pi) ↔ `UART2` (ESP32) | Отключи Bluetooth в `config.txt` для освобождения UART  |
| Скорость 	   | 921600 или 1.5 Мбит/с | Достаточно для 50-100 Гц телеметрии |
| Протокол 	   | Бинарный `struct` + синхробайты `0xAA55` + CRC16 | Устойчив к обрывам, минимальный оверхед |
| Библиотека на Pi | `python3-serial` или `C++: termios` | Выбор зависит от языка пайплайна |

### Внешняя связь (4G/5G HAT)
| Задача | Решение |
|--------|---------|
| Стриминг видео | `ffmpeg` → UDP/TCP → сервер или прямой приём на землю |
| Телеметрия | `MQTT` (через `paho-mqtt`) или сырой UDP |
| Управление | Обратный канал: команды с земли → 4G → Pi → UART → ESP32 |
| Резервирование | Локальная запись на SD, если 4G пропал |

## 🔄 ЭТАП 2: Переход на Dev Containers (когда будешь готов)

Когда код прототипа стабилизируется и тебе понадобится нативная отладка, автодополнение и работа без браузера:

1. **Останови Compose**: `docker compose down`
2. **Перемести Dockerfile**:
   ```bash
   mkdir -p .devcontainer
   mv Dockerfile.rpi-zero2w .devcontainer/Dockerfile
   ```
3. **Создай `.devcontainer/devcontainer.json`**:
   ```json
   {
     "name": "RPi Zero 2W Drone Dev",
     "build": { "dockerfile": "Dockerfile", "context": ".." },
     "remoteUser": "dev",
     "workspaceFolder": "/workspace",
     "customizations": {
       "vscode": {
         "extensions": ["ms-vscode.cpptools", "ms-python.python", "platformio.platformio-ide"],
         "settings": {
           "C_Cpp.default.compilerPath": "/usr/bin/aarch64-linux-gnu-gcc",
           "C_Cpp.default.cStandard": "c11",
           "C_Cpp.default.cppStandard": "c++17",
           "editor.formatOnSave": true
         }
       }
     },
     "postCreateCommand": "mkdir -p /workspace/archive && sudo chown -R dev:dev /workspace"
   }
   ```
4. **Запусти**: В VS Code `F1 → Dev Containers: Reopen in Container`

`docker-compose.yml` больше не нужен. Все файлы маппятся мгновенно, отладчик работает нативно.

## 📝 Что делаем внутри контейнера дальше?

Как только `http://localhost:8080` откроется:
1. Создаём папку `src/` с `CMakeLists.txt`, `main.cpp`, `uart_telemetry.h/cpp`
2. Пишем мок-камеру + приём UART-телеметрии
3. Тестируем логику оверлея и кодирования (без реального железа)
4. Когда получишь Pi Zero 2W → раскомментируем `devices:` в compose, заливаем бинарник, подключаем камеру и 4G-HAT

Привет! 👋 Ошибка `❌ code-server install failed` возникает потому, что **скрипт установки code-server не может скачать или распаковать бинарник для ARM64**. Это частая проблема с `--method=standalone` на `debian:bookworm-slim`.

Давай решим это **надёжным способом** — через прямую загрузку конкретного релиза с явным указанием архитектуры.

---

## 🔧 Исправленный `Dockerfile.rpi-zero2w` (гарантированно рабочий)

```dockerfile
# syntax=docker/dockerfile:1.4
# Dockerfile.rpi-zero2w
FROM debian:bookworm-slim

LABEL maintainer="your@email.com"
LABEL description="RPi Zero 2W dev env + code-server IDE"

ENV DEBIAN_FRONTEND=noninteractive \
    LANG=en_US.UTF-8 \
    LANGUAGE=en_US:en \
    LC_ALL=en_US.UTF-8 \
    CODE_SERVER_VERSION=4.92.2

# === 1. ВСЕ зависимости + кросс-компилятор + pkg-config + lib для code-server ===
RUN apt-get update && apt-get install -y --no-install-recommends \
    build-essential cmake git wget curl python3 python3-pip python3-serial \
    pkg-config gcc-aarch64-linux-gnu g++-aarch64-linux-gnu \
    libgpiod-dev sudo locales \
    # Зависимости для code-server (обязательно!)
    libsecret-1-0 libx11-6 libxkbfile1 libkrb5-3 ca-certificates \
    && rm -rf /var/lib/apt/lists/*

# === 2. Локаль ===
RUN sed -i 's/# en_US.UTF-8 UTF-8/en_US.UTF-8 UTF-8/' /etc/locale.gen && \
    locale-gen en_US.UTF-8

# === 3. Установка code-server ПРЯМОЙ ЗАГРУЗКОЙ (надёжнее install.sh!) ===
RUN ARCH=$(uname -m | sed 's/aarch64/arm64/;s/armv7l/armhf/') && \
    wget -qO- "https://github.com/coder/code-server/releases/download/v${CODE_SERVER_VERSION}/code-server_${CODE_SERVER_VERSION}_linux-${ARCH}.tar.gz" | \
    tar -xzC /opt && \
    ln -s /opt/code-server_${CODE_SERVER_VERSION}_linux-${ARCH}/bin/code-server /usr/local/bin/code-server && \
    which code-server && code-server --version

# === 4. Создание пользователя 'dev' (ПЕРЕД настройкой конфигов!) ===
RUN useradd -m -u 1000 -s /bin/bash dev && \
    echo "dev:dev" | chpasswd && \
    adduser dev sudo && \
    echo "dev ALL=(ALL) NOPASSWD:ALL" >> /etc/sudoers && \
    usermod -aG video,dialout dev && \
    mkdir -p /workspace /home/dev/.local/share/code-server/User && \
    chown -R dev:dev /workspace /home/dev/.local

# === 5. Переключаемся на dev ===
USER dev
WORKDIR /workspace

# === 6. Настройки VS Code (через printf — надёжнее heredoc) ===
RUN printf '%s\n' \
    '{' \
    '  "extensions.autoUpdate": false,' \
    '  "files.autoSave": "afterDelay",' \
    '  "editor.formatOnSave": true,' \
    '  "C_Cpp.default.compilerPath": "/usr/bin/aarch64-linux-gnu-gcc",' \
    '  "C_Cpp.default.cStandard": "c11",' \
    '  "C_Cpp.default.cppStandard": "c++17",' \
    '  "terminal.integrated.defaultProfile.linux": "bash",' \
    '  "files.encoding": "utf8"' \
    '}' > /home/dev/.local/share/code-server/User/settings.json

# === 7. Расширения (теперь code-server точно установлен) ===
RUN code-server --install-extension ms-vscode.cpptools --force || true && \
    code-server --install-extension platformio.platformio-ide --force || true

EXPOSE 8080
CMD ["code-server", "--bind-addr", "0.0.0.0:8080", "--auth", "none", "/workspace"]
```

```python
#!/usr/bin/env python3

import os, sys, time

def main():
    print("🚀 Drone Video Pipeline v0.1")
```
