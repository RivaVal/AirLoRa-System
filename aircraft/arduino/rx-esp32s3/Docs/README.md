# Arduino Motor Control Project


## Структура проекта
- `Eleron_Controller` - управление сервоприводами
- `ICM_Handler` - работа с датчиком ICM-20948
- `SD_Handler` - запись данных на SD карту
- `SPI_Managger` - manager по работе с SPI устройствами
- `motor_controller` - управление моторами 
- `E49_Controller` - управление связью проекта

/**
 * @file SystemArchitecture.md
 * @brief Архитектура системы БПЛА
 * 
 * СИСТЕМНЫЕ КОМПОНЕНТЫ:
 * 
 * 1. VSPI МЕНЕДЖМЕНТ (SPI_Manager)
 *    - Управление доступом к VSPI для SD карты и ICM-20948
 *    - Разрешение конфликтов через мьютексы
 *    - Комплексное тестирование шины
 * 
 * 2. РАДИОМОДУЛЬ E49 (E49_Controller)  
 *    - Прием/передача команд управления
 *    - Non-blocking операции
 *    - CRC проверка и ACK подтверждения
 * 
 * 3. ИНЕРЦИАЛЬНЫЙ ДАТЧИК (ICM_Handler)
 *    - Чтение акселерометра, гироскопа, магнитометра
 *    - Расчет углов Эйлера и кватернионов
 *    - DMP обработка (если доступно)
 * 
 * 4. СИСТЕМА ХРАНЕНИЯ (SD_Handler)
 *    - Буферизованная запись данных сенсоров
 *    - Автоматическое создание файлов
 *    - Мониторинг здоровья карты
 * 
 * 5. СИСТЕМА УПРАВЛЕНИЯ (UnifiedLEDCController)
 *    - Управление сервоприводами и моторами через LEDC
 *    - Комплексное тестирование исполнительных механизмов
 * 
 * VSPI КОНФИГУРАЦИЯ:
 *    SCK  - GPIO 18
 *    MISO - GPIO 19  
 *    MOSI - GPIO 23
 *    ICM_CS - GPIO 22
 *    SD_CS  - GPIO 15
 * 
 * ПРИОРИТЕТ VSPI:
 *    1. ICM-20948 (высший приоритет - навигация)
 *    2. SD Card (низший приоритет - логирование)
 */


1. Табличный CRC8
    Ускорение расчетов в 8 раз через lookup table
    Полином 0x07 (стандарт для通信)
2. ESP_ERROR проверки
    Проверка инициализации SPI
    Валидация всех критических операций
    Автоматическое восстановление при ошибках
3. Неблокирующая реализация
    Обновление по таймеру (50 Гц)
    Отслеживание таймаутов соединения
    Автоматическое восстановление приема
4. Улучшенная диагностика
    Детальная статистика работы
    Мониторинг качества связи (RSSI/SNR)
    Callback'и для событий соединения
5. Оптимизация для приемника
    Приоритет надежности приема над скоростью
    Автоматические повторные попытки
    Валидация всех полей пакета
Этот код обеспечит стабильный прием данных с частотой 5 Гц и 
автоматическое восстановление при потере связи, 
что критически важно для БПЛА.
