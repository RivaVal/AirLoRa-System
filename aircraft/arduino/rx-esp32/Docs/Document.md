Назначение модуля:
    Прием данных от удаленного Sender модуля
    Отправка подтверждений о получении (ACK)
    Контроль целостности данных
    Ведение статистики связи
Особенности реализации:
    Валидация пакетов - проверка преамбулы, ID и timestamp
    Статистика - подсчет принятых пакетов, ошибок, ACK
    Обработка ошибок - повторные попытки инициации
    Мониторинг - периодический вывод статуса
Настройки связи (должны совпадать с Sender!):
    Скорость UART: 9600 bps
    Скорость воздуха: 19200 bps
    Адрес: High=1, Low=101
    Канал: 17
    Мощность: 17 dBm
    FEC: Включено
Протокол связи:
    Sender отправляет данные с флагом ACK
    Receiver validates packet
    Если нужен ACK - Receiver отправляет подтверждение
    Sender получает ACK и обновляет статус
Диагностика:
    Проверка пина AUX перед операциями
    Подробное логирование всех событий
    Статистика каждые 10 секунд
    Авторестарт при критических ошибках

    Убедись в идентичности настроек на обоих модулях

1. Обзор Архитектуры и Применяемых Технологий
Ваш проект представляет собой систему беспилотного летательного аппарата (БПЛА) с приемником команд управления на базе ESP32. Он использует несколько технологий для связи, сенсорики и исполнительных механизмов:

ESP32: Микроконтроллер с Wi-Fi и Bluetooth, основной вычислительный элемент.
Среда разработки: Arduino IDE с поддержкой ESP-IDF v5.0+. Это позволяет использовать как высокоуровневые функции Arduino, так и низкоуровневые API ESP-IDF.
Платформа: Предположительно ESP32 DevKitC или аналогичная с 38 контактами.
LoRa (Long Range): Технология беспроводной связи на сверхдлинные расстояния с низким энергопотреблением.
Модуль: SX1278 (например, Ra-01).
Интерфейс: SPI (Serial Peripheral Interface).
SCK: Тактовый сигнал от ESP32 к LoRa.
MISO: Данные от LoRa к ESP32.
MOSI: Данные от ESP32 к LoRa.
NSS/SS: Выбор ведомого устройства (LoRa).
RST: Сброс LoRa модуля.
DIO0: Прерывание для уведомления ESP32 о событиях (например, приеме пакета).
Библиотека: RadioLib используется для удобного взаимодействия с LoRa модулем.
Назначение: Прием команд управления от удаленного передатчика.
UART (Universal Asynchronous Receiver/Transmitter):
Назначение: Связь с GPS модулем (ATGM336H).
Подключение: Перекрестное (TX одного устройства к RX другого).
GPS_TX (модуль) -> ESP32_RX (GPIO16).
ESP32_TX (GPIO17) -> GPS_RX (модуль).
I2C (Inter-Integrated Circuit):
Назначение: Связь с несколькими устройствами по двум проводам (SDA - данные, SCL - такт).
Шина: Общая I2C на GPIO21 (SDA) и GPIO22 (SCL).
Устройства:
IMU (GY-87, ICM-20948): Датчики навигации (гироскоп, акселерометр, магнитометр).
ESP32-CAM: Возможно, для передачи изображений или синхронизации.
Другие I2C устройства (EEPROM, I/O Expander).
PWM (Pulse Width Modulation) / LEDC (LED Control):
Назначение: Управление сервоприводами и моторами.
Технология: Используется аппаратный блок LEDC ESP32 для генерации точных ШИМ сигналов.
Пины: Сервы (GPIO 25, 26, 27, 15, 33), Моторы (GPIO 32, 14).
Модуль: UnifiedLEDCController - унифицированный класс для управления всеми ШИМ устройствами.
Конфигурация: Структура ModuleConfig позволяет включать/отключать поддержку различных модулей программно.
Структуры данных: DataComSet_t, AckPacket_t определяют формат данных, передаваемых по радиоканалу.

Основные Найденные Проблемы:

Однако в UnifiedLEDCController.cpp при вызовах ledc_set_duty, ledc_update_duty, setupLEDCChannel часто используется Config::Pins::servoChannels[servoNum] вместо channel_config.channel = Config::Pins::servoChannels[servoNum]; внутри setupLEDCChannel. 4. Проверка Алгоритма Работы и Настройки Приемника
Настройка LoRa: Убедитесь, что параметры (частота, SF, BW, CR) совпадают с настройками передатчика.
Цикл обработки: FastLoRaReceiver должен использовать неблокирующий подход (state machine или non-blocking calls), как и планировалось. Проверьте, что функция получения данных вызывается регулярно (например, в loop()).
Отправка ACK: Логика отправки подтверждения должна корректно срабатывать при установленном флаге ACK_REQUEST_FLAG и при получении уникального пакета.
Обновление ШИМ: После получения команды из DataComSet_t, значения должны передаваться в UnifiedLEDCController, который затем обновляет сигнал на соответствующих пинах.
Тестирование: Реализуйте или используйте встроенные механизмы тестирования (runSelfTest, updateServoTest, updateMotorTest) для проверки работы каждого модуля отдельно и в комплексе.
